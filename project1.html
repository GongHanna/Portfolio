Issue & Solution

1. SVG 색상이 왜 변경이 안될까?

  • Issue

    SVG 색상 변경을 위해 color 값을 주었지만 색상이 바뀌지 않음

  • Solution

    SVG의 색상을 변경하려면 SVG 자식이 가지고 있는
    <path>의 색상을 제어해야 함

    SVG 자식인 <path>에 class 명 부여

    <path>는 color가 아닌 fill로 색상 제어

2. GSAP? PulgIn? JS?

  • Issue

    스크롤시 특정 구간에서 발생하는 이벤트를 위해 기존에 배웠던
    ScrollTrigger에 대해서 공부

  • Solution

    scrollTrigger는 이 사이트에 필요 없다고 판단

    css로 돌아가서 position: sticky로 제어

    jQuery의 scroll 이벤트 메소드 사용

3. 애니메이션일까 로딩일까?

  • Issue

    페이지 내 이벤트 요소가 많아 어떤 이벤트 메서드를 사용해야
    그대로 구현이 가능할지 구분이 힘듦

  • Solution

    이벤트들을 정리하기 위해 노트에 이벤트 순서 필기

    load 이벤트 메소드 사용 및 addClass() 적용

    animation-delay 적용

Review

이 사이트 개인 프로젝트로 선정했을 때, '퍼블리싱 기술을 똑같이 구현하자!' 라는 목표를 잡았습니다.

하루에 목표한 것을 마무리 하지 못하거나 막힌 부분이 있다면 자기 전까지 계속 생각이 났습니다. ‘이 코드는 이런 코드로 작성해보면 어떨까?’

이벤트 적용에 있어서 막히는 부분이 생기면 원본 사이트에 들어가서 막힌 부분을 하염없이 보고 생각했습니다.
그리고 나서 적용 안 된 이벤트들을 노트에 작성하며 고민하고 공부했습니다.

몇 번의 html 구조 수정과 css 수정, 그리고 JavaScript의 다양한 이벤트들이 저를 괴롭게 만들었지만,
그 괴로움과 고뇌의 시간들이 모여 웹 사이트가 완성되었을 때의 쾌감과 즐거움은 잊을 수 없을 것 같습니다.

이번 경험을 통해 많은 것들을 공부하고, 좀 더 발전할 수 있게 된 좋은 기회였다고 생각합니다.
